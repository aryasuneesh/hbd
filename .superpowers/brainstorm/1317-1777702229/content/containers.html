Smooth Animation Strategy
All animations will use CSS transform + opacity only (GPU-composited, never layout-triggering properties). Spring physics via Framer Motion for natural deceleration — no linear easing. Each animation step triggered by a single state change, not chained timeouts. will-change: transform pre-promoted on animated elements to avoid paint during the sequence.
